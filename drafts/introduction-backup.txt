# Lorenz transformation in logic?

The space-time tradeoff Memoisation versus lazy evaluation.

Memoization (Time $\rightarrow$ Space)

You take a dynamic, temporal proof that requires many coinductive steps (Time), evaluate it in advance, and cache all the valid answers into a giant, static lookup table (Space/Induction). You have contracted logical time and dilated logical space.

Lazy Evaluation (Space $\rightarrow$ Time)

You take a massive, eagerly-evaluated structural memory footprint (Space/Induction) and collapse it. Instead of storing the data, you store a tiny coalgebric rule that generates the proofs sequentially, only exactly when requested (Time/Coinduction). You have contracted logical space and dilated logical time.

In computer science and proof theory, there is a fundamental principle called Confluence (or the Church-Rosser theorem). It states that if you have a valid computation, the path, perspective, or order in which you evaluate it does not change its final semantic truth.

Compact elements

A compact element (sometimes called a finite element). An element $c$ is compact if, whenever $c$ is less than or equal to the limit of a directed set, $c$ is actually less than or equal to some element already inside that directed set.
